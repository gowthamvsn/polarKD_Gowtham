What "Causal Assumptions and Rules" Means Here
Three distinct layers, each adding a different kind of rigour:

Layer A — Domain Assumptions (Hard Knowledge)
Pre-defined facts from climate science that override LLM decisions:

Direction rules — known one-way causality:


GHG emissions → temperature       (never reverse)
temperature → sea ice extent      (warming melts ice, not reverse)
sea ice → albedo                  (less ice = lower albedo)
albedo → radiative forcing        (lower albedo = more absorbed heat)
Forbidden edges — physically impossible directions:


sea ice → temperature             (ice doesn't warm the atmosphere)
precipitation → cloud cover       (effect can't cause its own cause)
Required edges — known links to always include regardless of LLM output:


CO2 concentration → surface temperature    (always causal, always this direction)
These live in a new causal_assumptions.py as configurable rule tables — no hardcoding in logic.

Layer B — Graph Structure Rules (Formal Causal Inference)
Rules that enforce structural properties of a valid causal graph:

DAG enforcement — detect cycles (A→B→C→A), break them by removing the lowest-confidence edge. A cycle means the LLM contradicted itself.

Transitivity marking — if A→B and B→C exist, mark A→C as an implied indirect path (shown differently in the graph, e.g. dashed edge).

Confounder detection — if variable X causes both Y and Z, flag X as a common cause. Useful for the user to know a spurious Y↔Z correlation may exist.

Collider detection — if both X and Z cause Y, flag Y as a collider node. These are important in causal inference (conditioning on them opens spurious paths).

Layer C — Algorithmic Causal Discovery (Statistical)
You already have PCMCI in your repo. This layer uses statistical methods to validate or score the LLM-extracted edges:

PC algorithm — constraint-based structure learning from the confidence scores
PCMCI — if your paper data has time series, causal links can be tested statistically (lag-aware)
Confidence re-scoring — use statistical independence test results to adjust LLM confidence scores
This is optional and computationally heavier, but very powerful if the underlying data is available.
